REPEAT, ENDREPEAT, and BREAK

Code between REPEAT and ENDREPEAT can be executed multiple times. Starting right after the REPEAT, each command is executed, and after the last command before the ENDREPEAT, the computer goes back to the first command in the REPEAT - ENDREPEAT "block." Commands like this that can be executed repeatedly are sometimes called a loop. IF a BREAK command is executed, then the computer goes straight to the command after the ENDREPEAT. Here are several examples:

VVBasic commands Output
CLEAR
PRINT "Square roots from 1 to 10"
NUMBER N = 1
REPEAT
  PRINT N, Round(Pow(N, 0.5), 3)
  IF N = 10 THEN
    BREAK
  ENDIF
  LET N = N + 1
ENDREPEAT
PRINT "Done."

CLEAR
PRINT "Guess my number!"
NUMBER MyNumber
REPEAT
  LET MyNumber = Random(1,5)
  REPEAT
    NUMBER YourGuess
    INPUT "Enter your guess:" YourGuess
    IF YourGuess = MyNumber THEN
      PRINT "That's it!"
      BREAK
    ELSE
      PRINT "No, try again."
    ENDIF
  ENDREPEAT
  TEXT PlayAgain
  INPUT "Play again? " PlayAgain
    WHEN PlayAgain <> "Y" AND
         PlayAgain <> "N"
    COMPLAIN "Please enter Y or N"
  IF PlayAgain = "N" THEN
    BREAK
  ENDIF
ENDREPEAT

Notice that in every example the BREAK command is inside an IF. That makes sense, because if the BREAK were not in an IF it would be executed on the first time through the loop, and there would be no repetition. BREAKs are almost always found inside an IF command. (And also inside a REPEAT - ENDREPEAT block -- it is an error to use BREAK anywhere else.)

Also notice that the lines between REPEAT and ENDREPEAT are indented. This is not required by the computer, but it is extremely helpful for making the program readable.

The third example above has one REPEAT - ENDREPEAT within another. Each loop has its own BREAK. The outer loop is executed once for each time the game is played, and the inner loop is executed once for each guess the player makes. Study this program carefully and make sure you understand it.
